# Reflection
## Decisions made
Here is a list of decisions I have made when completing this project:

1. Tools: I had to find tools that would increase efficiency. Copying and pasting code from ChatGPT was better than writing code by hand. BUT granting Codex permissions to actually modify files was a gamechanger. 
2. Priorities: I accepted that I was not going to get the realism I desired in the time frame of this class. This means that good, can good enough to ship. That helped limit the scope and make the project more achievable.
3. Listen to Feedback: I thought that I needed to signify strongly what to do when you got to the main page, but I did not. Based on peer feedback, users intuitively knew what to do when they got to the page, so a strong signifier was not necessary. This helped me save effort and time on the project.
* These decisions were made because I had limited time to work on these projects because I am a full-time employee and a full-time student.  

## What worked?
**Functional interaction: ** Adding the pullback racer worked better than I imagined it would be.  
**Tool adoption:** The adoption of Codex and allowing it access to my files helped me make changes much quicker and still let me review in real time before allowing those changes, and still stay with GitHub desktop for committing and pushing changes to GitHub. 

## What would I do differently? 
Here is a list of things that I **would** do differently if I had to do it again:

* Scripts: I noticed when reviewing another student’s page using developer tools that they had a script file separate from their index and just referenced the script.js file in their index. This would help keep my index file more organized. 
* Tool adoption: I wish I would not have been so resistant to using codex or another coding tool. It moved my project forward much faster. 

## What did I learn?

* I learned that sometimes when you start coding, the end result does not always turn out like you expect. I really wanted this project to turn out more realistic, but came to grow the cartoon look of some of the items on my desktop. I also learned that slow adoption of new technology could leave you behind, or at least that faster adoption can help you achieve goals more quickly.
